# باب 1: فزکس سیمولیشن کی بنیاد

## سیکھنے کے مقاصد

- روبوٹ سیمولیشن میں بنیادی فزکس کے تصورات کو سمجھیں
- روبوٹکس میں فزکس انجن کے اصول سیکھیں
- Gazebo کی فزکس سیمولیشن کی صلاحیتوں کا جائزہ لیں
- سیمولیشن میں گریویٹی، کولیژن، اور ڈائنیمکس کو سمجھیں

## فزکس سیمولیشن کا تعارف

فزکس سیمولیشن روبوٹکس کی ترقی کے لیے انتہائی اہم ہے کیونکہ یہ حقیقی ہارڈ ویئر پر ڈپلائمنٹ سے پہلے محفوظ، قیمت مؤثر ماحول میں روبوٹ کے رویوں کی ٹیسٹنگ کی اجازت دیتا ہے۔ ہیومینائیڈ روبوٹکس میں، درست فزکس سیمولیشن خاص طور پر اہم ہے کیونکہ بائی پیڈل لوکوموشن اور ماحول کے ساتھ تعامل میں پیچیدہ ڈائنیمکس شامل ہوتے ہیں۔

### ہیومینائیڈ روبوٹس کے لیے فزکس سیمولیشن کیوں اہم ہے

ہیومینائیڈ روبوٹس سیمولیشن میں منفرد چیلنجز پیش کرتے ہیں:

- **پیچیدہ ڈائنیمکس**: ہیومینائیڈ روبوٹس میں متعدد ڈگریز آف فریڈم اور پیچیدہ کنیمیٹک چینز ہوتے ہیں
- **بیلنس اور سٹیبیلٹی**: بیلنس برقرار رکھنے کے لیے درست کنٹرول اور فزکس کی سمجھ کی ضرورت ہوتی ہے
- **ماحول کے ساتھ تعامل**: چلنا، مینوپولیشن، اور نیویگیشن سبھی پیچیدہ فزکس انٹرایکشنز کو شامل کرتے ہیں

## فزکس انجن

فزکس انجن ایک سافٹ ویئر جزو ہے جو جسمانی سسٹمز کی شبیہ کاری کرتا ہے۔ روبوٹکس میں، فزکس انجن یہ کام انجام دیتے ہیں:

- کولیژن ڈیٹیکشن اور ریسپانس
- ریجیڈ باڈی ڈائنیمکس
- جوائنٹ کنٹرینٹس
- کنٹیکٹ فزکس
- ماس اور جامد خصوصیات

### کلیدی فزکس کے تصورات

1. **ریجیڈ باڈی**: ایک ٹھوس باڈی کی مثال جس میں ڈیفورمیشن کو نظر انداز کر دیا جاتا ہے
2. **ڈگریز آف فریڈم (DoF)**: آزاد حرکتوں کی تعداد جو باڈی کے پاس ہوتی ہے
3. **جوائنٹ کنٹرینٹس**: اس بات کی پابندیاں کہ باڈیز ایک دوسرے کے مقابلے میں کیسے حرکت کر سکتی ہیں
4. **کولیژن ڈیٹیکشن**: الگورتھم جو یہ تعین کرتا ہے کہ جب دو اشیاء ایک دوسرے کو کاٹتی ہیں
5. **کنٹیکٹ ریسپانس**: اشیاء کا رد عمل جب وہ ٹکراتی ہیں

## Gazebo فزکس سیمولیشن

Gazebo ODE (Open Dynamics Engine) فزکس انجن کو ڈیفالٹ کے طور پر استعمال کرتا ہے، ہر چند کہ یہ دیگر کی حمایت کرتا ہے۔ کلیدی خصوصیات میں شامل ہیں:

- ریجیڈ-بodies ڈائنیمکس کی درست شبیہ کاری
- مضبوط اور تیز کولیژن ڈیٹیکشن
- متعدد سینسرز کی شبیہ کاری (LIDAR، ڈیپتھ کیمرز، IMUs)
- حقیقی نظر آنے والی رینڈرنگ اور وژولائزیشن

### Gazebo فزکس پیرامیٹرز

سیمولیشن ماڈلز تخلیق کرتے وقت، آپ فزکس پیرامیٹرز کے ساتھ کام کریں گے جیسے:

- **ماس**: لنک کی ماس
- **جامد**: لنک کا گھومنے والا جامد
- **فرکشن**: اسٹیٹک اور ڈائنیمک فرکشن کوائف
- **ڈیمپنگ**: لکیری اور زاویہ وار ڈیمپنگ پیرامیٹرز
- **زیادہ سے زیادہ رفتار**: زیادہ سے زیادہ رفتار کی پابندیاں

## گریویٹی اور ماحولیاتی فزکس

Gazebo میں، آپ ماحولیاتی فزکس پیرامیٹرز کو ترتیب دے سکتے ہیں:

```xml
<!-- ایک دنیا کی فائل میں -->
<physics type="ode">
  <gravity>0 0 -9.8</gravity>
  <max_step_size>0.001</max_step_size>
  <real_time_factor>1</real_time_factor>
  <real_time_update_rate>1000</real_time_update_rate>
</physics>
```

## کولیژن ماڈلز

کولیژن ماڈلز کی مختلف اقسام ہیں:

- **کولیژن جیومیٹری**: یہ بیان کرتا ہے کہ اشیاء جسمانی طور پر کیسے تعامل کرتی ہیں
- **وژول جیومیٹری**: یہ بیان کرتا ہے کہ اشیاء کیسے نظر آتی ہیں (کولیژن جیومیٹری سے مختلف ہو سکتی ہے)
- **کنٹیکٹ سینسرز**: جب اشیاء رابطہ قائم کرتی ہیں تو یہ پتہ لگانے کے لیے استعمال کیے جاتے ہیں

## سیمولیشن کی درستی کے غور

جب سیمولیشن کے ساتھ ترقی کرتے ہو، غور کریں:

1. **ماڈل کی درستی**: آپ کا سیمولیشن ماڈل حقیقی روبوٹ سے کتنی قریب سے مماثلت رکھتا ہے
2. **فزکس پیرامیٹرز**: مناسب طریقے سے ٹیون کی گئی ماس، جامد، اور فرکشن ویلیوز
3. **ہارڈ ویئر کی پابندیاں**: یہ سمجھنا کہ حقیقی دنیا کی کارکردگی سیمولیشن سے مختلف ہو سکتی ہے
4. **نظامی اختلافات**: سیمولیشن اور حقیقت کے درمیان عام اختلافات

## بنیادی فزکس سیمولیشن کا سیٹ اپ

```xml
<!-- URDF کا مثالی ٹکڑا فزکس کی خصوصیات کے ساتھ -->
<link name="link_name">
  <inertial>
    <mass value="1.0"/>
    <origin xyz="0 0 0.5" rpy="0 0 0"/>
    <inertia ixx="1.0" ixy="0.0" ixz="0.0" iyy="1.0" iyz="0.0" izz="1.0"/>
  </inertial>
  <visual>
    <origin xyz="0 0 0" rpy="0 0 0"/>
    <geometry>
      <box size="1 1 1"/>
    </geometry>
  </visual>
  <collision>
    <origin xyz="0 0 0" rpy="0 0 0"/>
    <geometry>
      <box size="1 1 1"/>
    </geometry>
  </collision>
</link>
```

## خلاصہ

فزکس سیمولیشن روبوٹ کی ترقی کی بنیاد ہے، خاص طور پر چیلنجز والے ڈائنیمکس والے پیچیدہ ہیومینائیڈ روبوٹس کے لیے۔ فزکس انجن کے اصولوں کو سمجھنا اور فزکس پیرامیٹرز کو مناسب طریقے سے تشکیل دینا مؤثر روبوٹ کی ترقی کے لیے انتہائی اہم ہے۔

## مشقیں

1. بنیادی فزکس کے ساتھ ایک سادہ Gazebo دنیا تخلیق کریں
2. ایک باکس شامل کریں اور اس کی فزکس خصوصیات سیٹ کریں
3. Gazebo میں گریویٹی کی ترتیبات کے ساتھ تجربہ کریں
